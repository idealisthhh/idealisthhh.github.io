K8s 基础概念

发表于 2020-08-05 | 分类于 运维工具

K8s 基础概念

参考文章：https://www.infoq.cn/article/KNMAVdo3jXs3qPKqTZBw

一、What is Kubernetes

Kubernetes 的英文原意是 ”舵手“，container 的原意是 ”容器、集装箱“。我很喜欢参考文章中的比喻——如果说 container 是轮船上的集装箱，那么 Kubernetes 就是管理这些 container的智能化的轮船 。

二、核心功能

  1. 服务的发现与负载的均衡

    Kubernetes 具有业务负载检查的能力，它会检测业务的负载，如果某业务本身的 CPU 利用率过高或响应时间太长，K8s 可以对该业务进行扩容。

  2. scheduling，container 的自动装箱，由 K8s来做存储的编排，同时连接存储和容器的生命周期

    Kubernetes 可以把用户提交的容器放到 K8s 管理的集群的某一个台节点上。K8s 会在集群中找一台相对空闲的机器来放置该容器。

  3. 自动化 container 的恢复

    Kubernetes 具有节点健康检测功能，它会检测宿主机并自动发现出故障的宿主机或软件。同时 K8s 还会把运行在失败节点的容器进行自动迁移，迁移到一个正在健康运行的宿主机上，来完成集群的一个自动恢复。

  4. 应用的自动发布和回滚，以及应用相关的配置密文的管理

  5. 批量执行 job 类型任务

三、Kubernetes 的基础架构

典型的二层架构和 server-client 架构。其中 Master 为中央的管控节点，连接 Node 。

而用户只能通过 UI 、CLI 这些组件和 Master 进行连接，把命令或状态下发给 Master，Master接着将命令或状态下发给节点，实现最终的执行。

Master 架构

  • API Server: Kubernetes 中的所有组件都和该组件相连接。组件和组件之间不会有单独的连接存在。
  • Controller: 该组件用来实现对集群状态的一些管理。上文提到的 1、3 号功能都是由该组件实现的。
  • Scheduler: 调度器。功能 2 的实现依赖该组件。
  • ectd: 一个分布式的存储系统。其中放置着 API Server 需要的原信息。ectd 自身是一个高可用的系统，通过该系统保证了 Kubernetes 的 Master 组件的高可用性。

Node 架构

  • Storage Plugin: 存储操作依赖该插件。
  • Network Plugin: 网络操作所依赖的插件。
  • Kube-proxy: 真正完成 server 组网的组件。

组件间关联例子

用户通过 UI 或 CLI 提交一个 Pod 给 Kubernetes 进行部署：

  1. Pod 请求通过 UI 或 CLI 提交给 Kubernetes API Server
  2. API Server 将该信息写入它的存储系统 etcd 中
  3. Scheduler 组件通过 API Server 的 watch (notification) 机制得到信息: “有个 Pod 需要被调度”。
  4. Scheduler 在完成这次调度后会向 API Server 反馈: “这个 Pod 需要被调度到某个节点上”。
  5. API Server 接收到这次操作后，将这次结果写到 etcd 中，同时通知相应节点真正执行启动此次Pod。

2020 美国数学建模竞赛赛后总结

今年的美赛由于疫情的影响，特别的划分为了两周，我们小组参加的是第二周的比赛，选题有B，C，F三个。

选题

在准备阶段时，我们组是打算主攻B题，所以在比赛时就直接选择了B题。但在对题目进行了近一天的分析后，发现这道题和前几年的B题不太一样，这次的B题需要的工科基础过于专业，需要结构力学方面的知识（在我们组的判断下认为），不在我们组的能力范围内，所以我们组选择了临时换题。换到了C题。

题目

在亚马逊创建的在线市场中，它为客户提供了评估和回顾他们订单的机会。个人评级-称为“星级” –购买者可以使用1（低评分，低满意度）到5（高评分,高满意度）来表达自己的想法。此外，客户可以提交文本的消息，称为“评论” –表达有关产品的更多意见和信息。其他顾客可以对这些评论提供有用或没有帮助的评分（称为“帮助评分”）.协助他们自己的产品购买决策。公司使用这些数据对于参与的市场,参与的时间获取知识,赢得在产品设计功能选择的潜在成功.

阳光公司计划在网上市场推出和销售三种新产品：微波炉，婴儿奶嘴和吹风机。他们雇用了您的团队为顾问通过过去的客户评分,确定过去客户的关键模式，关系，度量和参数，以及其他竞争产品相关的评分和评论，以：1）在线销售策略； 2）确定能提供产品理想性的潜在重要设计功能。SunShine公司过去曾使用数据为销售策略提供信息，但他们以前没有使用过这种特殊的组合和数据类型。Sunshine特别感兴趣的是这些数据中基于时间的模式，以及它们是否以某种方式的交互能帮助公司制作成功的产品。

为了协助你的这个项目,SunShine的数据中心已经提供了3数据份文件: hair_dryer.tsv, microwave.tsv, and pacifier.tsv.这些数据代表着顾客在指定时间对于三种产品评分和评论,对于数据标签定义的表单也提供.给你提供的数据文件是您解决此问题的唯一的数据。

要求:

1.分析提供的三个产品数据集，(以识别，描述和有数学支持证据，定量或定性，关系，星级，评论,和评分之内和之间的度量和参数，?)将帮助Sunshine Company在三个新的在线市场产品中取得成功。

2.使用您的分析解决以下来自Sunshine的特定问题和要求:

公司市场总监：

a. 为了阳光公司跟踪, 一旦这三种产品在网上销售,找到一种根据最能提供信息的评级和评论的数据量度.

b. 识别并讨论每个数据集中可能基于时间的度量和模式表示产品在在线市场中的声誉正在上升或下降。

c. 确定最佳的基于文本的度量和基于评级的度量的组合表示可能成功或失败的产品.

d. 特定星级会引起更多评论吗？ 例如，客户是否更有可能看到一系列低星级评定后写一些类型的评论？

e. 是基于文字的评论的特定质量描述，例如“热情’’“失望”以及其他词汇,与评分水平密切相关？

3.写一两页的信给阳光公司的市场总监，总结一下您团队的分析和结果。 包括针对您的团队结果的最有信心地的特定理由,推荐给市场总监。

思路

对评论文本进行情感分析，得到情感分。统计文本词汇量，与评分一起评估反馈的价值。通过分析反馈的数量和质量分析产品的口碑。

缓冲区炸弹

原理

​ C对于数组引用不进行任何边界检查，而且局部变量和状态信息（例如保存的寄存器值和返回地址）都存放在栈中。这两种情况结合到一起就能导致严重的程序错误，对越界的数组元素的写操作会破坏存储在栈中的状态信息。当程序使用这个被破坏的状态，试图重新加载寄存器或执行ret指令时，就会出现很严重的错误。
​ 一种特别常见的状态破坏称为缓冲区溢出(bufferoverflow)。

IDA Pro 入门

IDA Pro入门指南

IDA 文件打开界面

  1. IDA View-A是反汇编窗口
  2. HexView-A是十六进制格式显示的窗口
  3. Imports是导入表（程序中调用到的外面的函数）
  4. Functions是函数表（这个程序中的函数）
  5. Structures是结构
  6. Enums是枚举
    我们进行逆向分析的主要场所就在IDA View-A反汇编窗口

寄存器

  1. EAX 是”累加器”(accumulator), 它是很多加法乘法指令的缺省寄存器
  2. EBX 是”基地址”(base)寄存器, 在内存寻址时存放基地址
  3. ECX 是计数器(counter), 是重复(REP)前缀指令和LOOP指令的内定计数器
  4. EDX 则总是被用来放整数除法产生的余数
  5. ESI/EDI 分别叫做”源/目标索引寄存器”(source/destination index),因为在很多字符串操作指令中， DS:ESI指向源串,而ES:EDI指向目标串
  6. EBP 是”基址指针”(BASE POINTER), 它最经常被用作高级语言函数调用的”框架指针”(frame pointer)
  7. ESP 专门用作堆栈指针，被形象地称为栈顶指针，堆栈的顶部是地址小的区域，压入堆栈的数据越多，ESP也就越来越小。在32位平台上，ESP每次减少4字节。

生成汇编语句对应的机器代码

“Options” –> “General” –> “Disassmbly”

将右侧的”Number of opcode bytes” 修改为一个非零数即可显示程序对应机器码

“Edit” –> “Patch Program” –> “Assemble”

输入相应汇编代码后确定显示对应机器码

Ubuntu安装历程及所遇到的问题记录(N卡)

  • 安装方式：U盘启动器
  • 所用载体：128G移动固态硬盘
  • 安装版本：18.04(LTS)
  • 电脑型号：机械革命Z2(Nvidia显卡)

安装前设置

  1. 将固态格式化为ext4格式，大小选择4096字节。(所谓的4k对齐) (推荐软件：DiskGenius)

  2. 分区：分出三个分区（我的分配方案，8g+60g+50g）

  3. 进入BIOS，将 security boot 改为[false]

  4. 设置Root，将 UEFI USB DISK 项（指向你的U盘）放到第一，将UEFI HDD WINDOWS 项置后，防止其优先启动。

  5. 保存退出。

安装过程

  1. 由于有独显（N卡）存在，直接选择 Install Ubuntu 项会有驱动未安装导致界面卡住的问题。应将光标指向 “Install Ubuntu” 项（切勿选择），然后按 ‘e’ 键。在出现的界面中找到 quiet splash，将其改为 nomodeset，之后按 F10 进入安装。

  2. 根据需要选择语言和键盘分布等，到”安装选项“项时又要注意的地方。

  3. 当安装进程达到”安装选项”时，选择其他选项，点击 继续。

  4. 关键部分分区，在栏中找到空闲项，将8g分区用于swap分区；将60g分区用于ext4日志文件系统，挂载于/分区；将50g分区用于ext4日志文件系统，挂载于/home分区。安装启动引导器的设备选择为该移动固态硬盘。点击 现在安装。（若界面中没有该选项，可按住 windows 键用鼠标拖动屏幕）

  5. 跟着流程安装，安装完后需要重启，在重启过程中要拔掉U盘，同时狂按 F10。(注：其他机器可能不是F10键，但是一定要进入grub菜单中)

安装结束后

  1. 进入以后，选择Ubuntu选项。之后会有紫框，光标指向 Ubuntu(切勿选择)，然后按 e 键，在出来的界面中找到 quiet splash，在其后添加 acpi_osi=Linux nomodeset，按 F10 键进入。

三种种标记语言

XML语言、 JSON语言、 YAML语言

XML语言

扩展标记语言(Extensible Markup Language, XML)，用于标记电子文件使其具有结构性的标记语言，可以用来标记数据、定义数据类型，是一种允许用户对自己的标记语言进行定义的源语言。
类似HTML，格式统一，跨平台和语言，早已成为业界公认的标准。
XML是标准通用标记语言 (SGML) 的子集，非常适合Web传输。XML提供统一的的方法来描述和交换独立于应用程序或供应商的结构化数据。
示例:

XML语言的优缺点

  • 优点：
    • 格式统一，符合标准
    • 容易与其他系统进行远程交互，数据共享比较方便。
  • 缺点：
    • XML文件庞大，文件格式复杂，传输占带宽。
    • 服务器端和客户端都需要花费大量代码来解析XML，导致服务器端和客户端代码变得异常复杂且不易维护
    • 客户端不同浏览器之间解析XML的方式不一致，需要重复编写很多代码；
    • 服务器端和客户端解析XML花费较多的资源和时间。

JSON语言

JSON(JavaScript Object Notation, JSON)，一种轻量级的数据交换格式，具有良好的可读和便于快速编写的特性。可在不同平台之间进行数据交换。常用来完成配置文件，使用有类型键值对。
示例：

JSON语言的优缺点

  • 优点：
    • 数据结构比较简单，易于读写，格式是压缩的，占用的带宽小；
    • 易于解析，客户端JavaScript可以简单的通过eval()进行JSON数据的读取；
    • 支持多种语言，包括ActionScript, C, C#, ColdFusion, Java, JavaScript, Perl, PHP, Python, Ruby等服务器端语言，便于服务器端的解析；
    • 因为JSON格式能直接为服务器端代码使用，大大简化了服务器端和客户端的代码开发量，且完成任务不变，并且易于维护。
  • 缺点：
    • 没有XML格式这么推广的深入人心和喜用广泛，没有XML那么通用性；
    • JSON格式目前在Web Service中推广还属于初级阶段。

YAML语言

YAML(YAML Ain’t Markup Language, YAML)，使用无类型键值对。用减号表示并列关系，使用缩进来表示层次（这点和Python语言很像）。

YAML语言的优缺点

  • 优点：
    • 便于人类读写；
  • 缺点：
    • 类似于JSON语言。
